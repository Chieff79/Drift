# Установка Drift на роутер Keenetic

Руководство для пользователей, которые хотят защитить **все устройства** домашней сети одним подключением — без установки приложений на каждый телефон, ноутбук и Smart TV.

> Технические команды и названия пунктов меню в этом руководстве приведены на английском и русском языках — Keenetic переключается между интерфейсами в настройках учётной записи администратора.

## Что это даст

Когда Drift работает **на роутере**, вы получаете три практических преимущества:

1. **Защита всего дома за один раз.** Ноутбук, два смартфона, телевизор, игровая консоль, умная колонка, робот-пылесос — всё это ходит в интернет через защищённый тоннель, без установки чего-либо на самих устройствах. Вам не нужно помнить, «включено ли Drift на iPad у ребёнка».
2. **Чистый клиент.** На самом устройстве (телефон, ноутбук) нет ни приложения Drift, ни системного VPN-профиля. Для любых приложений картина мира выглядит так, будто вы обычным образом подключены к домашнему Wi-Fi. Это удобно, когда часть приложений не любит, что у устройства поднят системный VPN.
3. **Экономия батареи и процессора.** Шифрование и маршрутизация происходят на роутере, а не на телефоне. Особенно заметно на старых смартфонах.

Подходит для:

- семей с несколькими устройствами,
- квартир с умным домом,
- тех, кто хочет защитить гостевой Wi-Fi для приходящих.

## Совместимые модели Keenetic

Drift работает на роутерах Keenetic с встроенной поддержкой **WireGuard** (компонент доступен в `Общие настройки → Обновление компонентов`). Минимальная версия прошивки — **KeeneticOS 3.7**, рекомендуемая — **3.9 и выше**.

| Модель | Артикул | Статус |
|--------|---------|--------|
| Keenetic Giga | KN-1011 | OK |
| Keenetic Ultra | KN-1811 | OK, с запасом по производительности |
| Keenetic Hopper | KN-3810 | OK, лучший выбор по цене/скорости |
| Keenetic Extra | KN-1713 | OK для 100 Мбит/с |
| Keenetic Viva | KN-1912 | OK |
| Keenetic Speedster | KN-3010 | OK |
| Keenetic Hero 4G | KN-2310 | OK |
| Keenetic Air | KN-1613 | Ограниченно — шифрование до 40–50 Мбит/с |
| Keenetic 4G (KN-1211 и раньше) | — | Не рекомендуется, слабый процессор |

> Практическое правило: если в роутере нет аппаратного ускорения AES/ChaCha20, скорость через тоннель будет ограничена примерно 40–60 Мбит/с. Giga, Ultra, Hopper спокойно держат 300+ Мбит/с.

Узнать модель можно на наклейке под роутером или в web-интерфейсе в разделе **Системный монитор**.

## Что понадобится

- Активная подписка **Drift Premium+** (на младших тарифах экспорт WireGuard-конфига недоступен).
- Файл конфигурации `drift-keenetic.conf`, полученный у поддержки.
- Доступ к web-интерфейсу роутера (по умолчанию `http://192.168.1.1` или `http://my.keenetic.net`).
- Пароль администратора роутера. При первой настройке Keenetic просит его задать самостоятельно — если вы его не помните, придётся сбросить роутер (кнопка `Reset` 10+ секунд) и заново пройти мастер.

## Шаг 1. Получить WireGuard конфиг у поддержки Drift

1. Напишите в Telegram на [@driftvpn](https://t.me/driftvpn) сообщение: «Нужен конфиг для роутера Keenetic». Укажите предпочитаемую локацию (NL, KZ, US).
2. Служба поддержки пришлёт файл `.conf` в формате WireGuard. Содержимое похоже на:

```
[Interface]
PrivateKey = ...
Address    = 10.10.0.42/32
DNS        = 1.1.1.1, 1.0.0.1

[Peer]
PublicKey           = ...
AllowedIPs          = 0.0.0.0/0
Endpoint            = nl-edge-01.drift-vpn.example:51820
PersistentKeepalive = 25
```

3. Сохраните файл на компьютере, с которого будете настраивать роутер.

> Конфиг привязан к вашей подписке, не передавайте его третьим лицам.

## Шаг 2. Войти в web-интерфейс Keenetic

1. Подключитесь к роутеру по Wi-Fi или кабелем.
2. Откройте в браузере `http://192.168.1.1` или `http://my.keenetic.net`.
3. Введите логин (`admin` по умолчанию) и пароль, заданный при первой настройке.

Если видите мастер первой настройки — пройдите его до конца, убедитесь, что интернет на WAN-интерфейсе работает, затем возвращайтесь сюда.

![screenshot](img/keenetic-step-2.png)

## Шаг 3. Добавить WireGuard-подключение и загрузить конфиг

1. В левом меню: **Интернет → Другие подключения** (*Internet → Other connections*).
2. В блоке **WireGuard** нажмите **Добавить подключение** (*Add connection*).
3. Введите имя соединения: `Drift`.
4. Нажмите **Загрузить из файла** (*Load from file*) и выберите `drift-keenetic.conf`.
5. Keenetic автоматически заполнит поля: адрес интерфейса, публичный ключ пира, endpoint, AllowedIPs.
6. Опция **Использовать для выхода в Интернет** (*Use for internet access*) — **включите**, если хотите, чтобы весь трафик шёл через Drift. Если планируете маршрутизировать только часть устройств — оставьте выключенной и настройте правила на Шаге 4.
7. Нажмите **Сохранить**.

Если соединение поднялось — в списке появится зелёный индикатор, и будет видна принятая/отправленная статистика.

![screenshot](img/keenetic-step-3.png)

## Шаг 4. Настроить маршрутизацию (какие устройства идут через Drift)

Keenetic позволяет выбирать, какие устройства отправлять в какой канал.

1. Перейдите в **Сетевые правила → Приоритеты подключений** (*Network Rules → Connection Priorities*).
2. Перетащите `Drift` вверх, чтобы он был основным каналом, **или** оставьте основным WAN и назначьте `Drift` как альтернативный.
3. Откройте **Список устройств** (*My Networks and Wi-Fi → Device List*).
4. Для каждого устройства, которое должно ходить через Drift, откройте карточку устройства и в поле **Политика доступа в Интернет** (*Internet Access Policy*) выберите `Drift`.
5. Для устройств, которые должны остаться на обычном канале (например, банк-клиент на рабочем ноутбуке), оставьте политику по умолчанию.

Политики можно создавать отдельно: **Сетевые правила → Политики доступа** (*Network Rules → Internet Access Policies*) — удобно, если у вас несколько каналов.

## Шаг 5. Автоподключение при сбоях (Interface State Monitoring)

Чтобы Keenetic сам восстанавливал тоннель при обрыве:

1. Откройте карточку подключения `Drift` (Интернет → Другие подключения → Drift).
2. Найдите пункт **Проверка доступности интернета** (*Internet availability check*).
3. Включите **Ping Check** с адресом `1.1.1.1` или `8.8.8.8`, интервал — 30 секунд, порог — 3 неудачи.
4. Если проверка провалится, Keenetic автоматически переподнимет WireGuard-интерфейс.
5. В **Системные настройки → Планировщик** (*System → Scheduler*) можно добавить ночной reboot соединения раз в сутки — профилактика.

## Диагностика частых проблем

**Индикатор подключения красный, трафик не идёт.**
Проверьте, что часы на роутере идут правильно — WireGuard требует точного времени. `Общие настройки → Системное время → Синхронизировать с интернетом`.

**Скорость сильно меньше, чем без Drift.**
- Модель роутера не тянет шифрование — см. таблицу выше.
- В настройках подключения включите *Фрагментация MTU* = `1400`.
- Проверьте, не включён ли IPv6 WAN параллельно с IPv4 — в некоторых сценариях Keenetic пробует сначала IPv6 и теряет пакеты.

**Отдельные устройства жалуются на «нет интернета», хотя остальные работают.**
В карточке устройства временно переключите политику обратно на WAN и проверьте, что именно Drift блокирует. Часто причина — приложение на устройстве само использует VPN/DNS-over-HTTPS и конфликтует.

**Не открывается конкретный сайт (например, банковское приложение).**
Banking apps часто не любят зарубежные IP. Перенесите соответствующее устройство на обычный WAN через политики.

**Keenetic не видит компонент WireGuard.**
`Общие настройки → Обновление компонентов → Изменить набор компонентов` — убедитесь, что отмечен `WireGuard VPN`. Перезагрузите роутер после обновления.

## FAQ

**Можно ли использовать Drift на роутере и в приложении Drift на телефоне одновременно?**
Можно, но нет смысла — трафик будет шифроваться дважды, скорость упадёт. Если телефон дома — пусть ходит через роутер. Если вышли на улицу — включайте приложение Drift.

**Что с гостевым Wi-Fi?**
В Keenetic для гостевой сети можно выбрать отдельную политику — например, пускать гостей только через Drift, а домашние устройства через WAN. Настраивается в карточке сегмента сети.

**Можно ли поменять локацию без смены конфига?**
Нет, каждый конфиг привязан к конкретному узлу. Запросите дополнительный конфиг у поддержки и добавьте его вторым WireGuard-подключением — переключаться можно одной кнопкой.

**Где я могу посмотреть, сколько трафика прошло через Drift?**
`Интернет → Другие подключения → Drift → Статистика` — Keenetic ведёт график за 30 дней.

**Роутер перезагрузился — нужно ли снова загружать конфиг?**
Нет, все настройки сохраняются во flash.

**У меня два провайдера — можно ли балансировать?**
Да, Keenetic поддерживает multi-WAN. В приоритетах поставьте оба провайдера, поверх обоих — `Drift`. При падении одного провайдера тоннель автоматически переподнимется поверх второго.
